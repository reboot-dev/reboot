# PubSub

A pub/sub implementation that allows you to publish to `Topic`s and subscribe
[`Queue`](/library_services/queue)s to those `Topic`s.

Each `Topic` is associated with a unique ID and is created implicitly the
first time it is [`publish`](#publish)ed to or [`subscribe`](#subscribe)d to.

Messages are marshalled as [`Item`](/library_services/item)s to be published
to `Topic`s. They can be [`publish`](#publish)ed as a single `Value`, `bytes`,
or `Any` object or can be [`publish`](#bulk-publish)ed in bulk as a list of `Item`s.
In practice, you will want to use just one of these formats for your messages
in a given `Topic`. Read more about [`Item`](/library_services/item)s
to determine the best format for your data.

Subscription is [`Queue`](/library_services/queue) based. That is, when you
[`subscribe`](#subscribe) to a `Topic`, you reference a `Queue` by its ID.
Any messages published to that `Topic` will be received on the specified `Queue`
and can be [`dequeue`](/library_services/queue#dequeue)d to be read.


## Imports and servicers

To use the pub/sub library, import the library where you would like to use it.
To read messages from a subscribed [`Queue`](/library_services/queue), you will also need to import the
[`Queue`](/library_services/queue) library.

### Subscribe

Subscribe a single [`Queue`](/library_services/queue) as referenced by its ID to a `Topic`.
Any subsequent messages published to the `Topic` will be received by
[`dequeue`](/library_services/queue#dequeue)ing on the [`Queue`](/library_services/queue).

Note: Any messages published to the `Topic` *before* the `Queue` is subscribed
is not guaranteed to be received by the `Queue`.

### Bulk Publish

Publish a list of messages, formatted as [`Item`](/library_services/item)s.

This example demonstrates how to store different types of data in `Value`, but
in practice, you will want to use a uniform structure for all `Value`s
published to a `Topic`.

Note: The ability to add an `Item` with an `Any` format is only supported in Python, not TypeScript.

<Tabs groupId="language">
  <TabItem value="python" label="Python" default>
<!-- MARKDOWN-AUTO-DOCS:START
(CODE:src=../../../tests/reboot/std/pubsub/v1/pubsub_tests.py&lines=148-183) -->
<!-- The below code snippet is automatically added from ../../../tests/reboot/std/pubsub/v1/pubsub_tests.py -->

```py
from reboot.protobuf import (
    from_bool,
    from_dict,
    from_int,
    from_list,
    from_str,
    pack,
)
from reboot.std.item.v1.item import Item

await first_topic.publish(
    context,
    items=[
        Item(value=from_bool(True)),
        Item(value=from_int(3)),
        Item(value=from_str("apple")),
        Item(value=from_list(["a", "b", "c"])),
        Item(value=from_dict({"details": "details-go-here"})),
    ],
)

await second_topic.publish(
    context,
    items=[
        Item(bytes=b"some-bytes"),
        Item(bytes=b"some-more-bytes"),
    ],
)

await third_topic.publish(
    context,
    items=[
        Item(any=pack(any)),
        Item(any=pack(any)),
    ],
)
```
