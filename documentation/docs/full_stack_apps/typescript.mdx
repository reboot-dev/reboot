<Step
  title="Define your durable state data types and operations"
  description={`

First, import Zod via \`import { z } from "zod/v4"\`. Reboot requires
\`zod/v4\`.

Next, define a durable state data type in Reboot called \`ChatRoom\` with
\`state\` that includes an array of \`string\` messages.

For brevity we are defining just two operations:

1. \`messages\` returns all of your messages. It is a [\`reader\`](/learn_more/define/methods#kinds) method because all it does is read state.
1. \`send\` takes a \`message\` and updates state, so it is a [\`writer\`](/learn_more/define/methods#kinds) method.

&nbsp;

:::tip
Methods of kind [\`reader\`](/learn_more/define/methods#kinds) will safely be executed concurrently!
:::

`}>

```typescript title=./api/chat_room/v1/chat_room.ts
// ./api/chat_room/v1/chat_room.ts
import { reader, writer } from "@reboot-dev/reboot-api";
import { z } from "zod/v4";

export const ChatRoom = {
  state: {
    messages: z
      .array(z.string())
      .default(() => [])
      .meta({ tag: 1 }),
  },
  methods: {
    messages: reader({
      request: {},
      response: {
        messages: z.array(z.string()).meta({ tag: 1 }),
      },
    }),
    send: writer({
      request: {
        message: z.string().meta({ tag: 1 }),
      },
      response: {},
    }),
  },
};

export const api = {
  ChatRoom,
};
```

<Step
  title={<div>Implement <code>ChatServicer</code></div>}
  description={`

You can implement your servicer using either the class-based syntax or the
object literal syntax.

If class-based, your servicer must extend \`ChatRoom.Servicer\`.

If using the object literal syntax, set your servicer equal to
ChatRoom.servicer({ ... }).

If using the class-based syntax, an instance of will be created for every
instance of the
\`ChatRoom\` type that you create, and you can access the state of that
instance via \`this.state\`.

In the object literal syntax, you can access the state via the \`state\` variable
passed into the function.

Each method expects a \`context\` as the first argument that aligns
with the kind given in the Zod schema; \`messages\`
takes a \`ReaderContext\` and \`send\` takes a \`WriterContext\`.

\`send\` can mutate \`this.state\` /\ \`state\` and \`messages\` can only read from
it.

`}>

  <Tabs groupId="typescript-object-class">
    <TabItem value="object" label="Object literal" default>
<!-- prettier-ignore-start -->
```typescript title=./backend/src/chat_room_servicer.ts
// ./backend/src/chat_room_servicer.ts
import { ReaderContext, WriterContext } from "@reboot-dev/reboot";
import { ChatRoom } from "../../api/chat_room/v1/chat_room_rbt.js";

export const chatRoomServicer = ChatRoom.servicer({
  messages: async (
    context: ReaderContext,
    state: ChatRoom.State,
    request: ChatRoom.MessagesRequest
  ): Promise<ChatRoom.MessagesResponse> => {
    return { messages: state.messages };
  },

  send: async (
    context: WriterContext,
    state: ChatRoom.State,
    request: ChatRoom.SendRequest
  ): Promise<[ChatRoom.State, ChatRoom.SendResponse]> => {
    state.messages.push(request.message);
    return [state, {}];
  },
});
```

```typescript title=./backend/src/chat_room_servicer.ts
// ./backend/src/chat_room_servicer.ts
import { ReaderContext, WriterContext } from "@reboot-dev/reboot";
import { ChatRoom } from "../../api/chat_room/v1/chat_room_rbt.js";

export class ChatRoomServicer extends ChatRoom.Servicer {
  async messages(
    context: ReaderContext,
    request: ChatRoom.MessagesRequest
  ): Promise<ChatRoom.MessagesResponse> {
    return { messages: this.state.messages };
  }

  async send(
    context: WriterContext,
    request: ChatRoom.SendRequest
  ): Promise<ChatRoom.SendResponse> {
    this.state.messages.push(request.message);
    return {};
  }
}
```

<Step
  title={<div>Add necessary flags to your <code>.rbtrc</code> file</div>}
  description={`

First, tell Reboot where the Zod schemas live, the \`api/\` directory,
and where to output the generated code, also in \`api/\`.

Next, tell Reboot how to run your app. Give the app a name via
\`--name=chat-room\` to persist state between app restarts. Finally, tell Reboot
that it should use Node.js to run
our app.

:::tip
If you want to remove all of your changes between app restarts, do not pass the
\`--name\` flag to \`rbt\`.
:::

Last, specify the entrypoint for your app.

See a complete list of supported flags by running \`rbt dev run --help\`.

`}>

```bash
# .rbtrc
generate api/
generate --nodejs=api/

dev run --name=chat-room
dev run --watch=backend/src/**/*.ts
dev run --nodejs
dev run --application=backend/src/main.ts
```

</Step>
